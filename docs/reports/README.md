## ğŸ¯ **Organization Categories**

### **1. Project Status** (`project_status/`)
Reports about overall project status, readiness, and comprehensive project assessments.

**When to use:** For high-level project status reports, readiness assessments, and comprehensive project evaluations.

### **2. Refactoring & Cleanup** (`refactoring_cleanup/`)
Reports about code refactoring, cleanup efforts, workspace organization, and file organization.

**When to use:** For reports about refactoring work, cleanup operations, workspace organization, and code structure improvements.

### **3. Feature Implementation** (`feature_implementation/`)
Reports about specific feature implementations, feature completions, and feature-related fixes.

**When to use:** For reports documenting the implementation of specific features, feature completions, and feature-specific fixes.

### **4. Code Quality & Testing** (`code_quality_testing/`)
Reports about code quality improvements, testing efforts, and quality assurance work.

**When to use:** For reports about code quality fixes, testing summaries, and quality assurance work.

### **5. Session & Progress** (`session_progress/`)
Reports about individual work sessions, progress updates, and session summaries.

**When to use:** For reports documenting individual work sessions, progress updates, and session completion summaries.

### **6. Optimization** (`optimization/`)
Reports about performance optimizations, system optimizations, and optimization efforts.

**When to use:** For reports about performance improvements, system optimizations, and optimization work.

## ğŸ“‹ **File Naming Conventions**

Reports should follow these naming patterns:

- **Feature Implementation:** `[feature_name]_implementation_YYYY-MM-DD.md`
- **Project Status:** `[status_type]_REPORT_YYYY-MM-DD.md` or `[status_type]_SUMMARY.md`
- **Refactoring:** `[refactoring_type]_REPORT_YYYY-MM-DD.md` or `[refactoring_type]_SUMMARY.md`
- **Session Reports:** `SESSION_[type]_YYYY-MM-DD.md` or `PROGRESS_REPORT_YYYY-MM-DD.md`
- **Quality/Testing:** `[quality_type]_SUMMARY.md` or `[test_type]_SUMMARY.md`
